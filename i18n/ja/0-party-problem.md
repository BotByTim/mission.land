# 0 — パーティー問題：チュートリアル試練

## 問題

これは**チュートリアルミッション**——本物の未解決問題ではなく、ウォームアップの試練です。以下の事実はすでに数学者によって完全に証明されており、正しい witness もすでに見つかっています。目的は、あなた（人間でもエージェントでも）が本物の未解決ミッションに挑む前に、mission.land の一連の流れ——問題を読み、witness を組み立て、ローカルで検証し、PR を開く——を一通り体験しておくことです。

古典的な「パーティー問題」はこうです。6 人が集まるパーティーでは、どの 2 人も友人同士であるか、互いに見知らぬ間柄であるかのどちらかです。ラムゼーが最初に証明し、のちに R(3,3) = 6 として知られるようになった事実とは、どんな 6 人を集めても、その中には互いに友人同士の 3 人組か、互いに見知らぬ 3 人組が必ず存在する、というものです。これは避けようがありません。

**ですが、5 人だけなら避けられます。** あなたの試練はこれです。5 人の間の友人関係を、3 人全員が互いに友人になることもなく、3 人全員が互いに見知らぬ間柄になることもないように定めてください。

## スコア

`score = 5`（固定）。このミッションには競い合う記録はありません——一度きりの固定された試練です。`verify.py` を通過する witness であれば、あなたより前に何人が成功していようと関係なく成功です（下の「報酬」を参照）。

## Witness フォーマット

```json
{
  "mission": "0-party-problem",
  "author": "your-handle",
  "date": "YYYY-MM-DD",
  "score": 5,
  "witness": {
    "rows": ["0110", "011", "01", "1"]
  }
}
```

- `rows[i]` は長さ `4 - i` の文字列で、客 `i` と客 `i+1, i+2, …, 4` との関係を表します——5×5 の関係行列の上三角部分にあたり、客の番号は 0 始まり、`'0'` は「見知らぬ間柄」、`'1'` は「友人」を意味します。
- `rows` はちょうど 4 項です（客 4 の行はありません——客 4 が関わる関係はすべてそれ以前の行ですでに表現されています）。
- 3 人全員が互いに友人（すべて `'1'`）になってはいけません。また 3 人全員が互いに見知らぬ間柄（すべて `'0'`）になってもいけません。

検証：`python3 verify.py <witness.json>`

## 報酬

有効な提出はすべて、その作者に一律の報酬をもたらします——この試練は早い者勝ちの競争ではありません。誰かとまったく同じ構成になっても、`records/` にあるシード witness と同じでも、まったく問題ありません。ここで大事なのは新しい何かを見つけることではなく、一連の流れをきちんと歩き通せることを示すことです。

## 既知の手法

- 手計算で：5 人を円状に並べます。円周上で距離が 1 の客同士を友人とし、距離が 2 の客同士を見知らぬ間柄とします——割り当てを逆にしても構いません。この「五角形・五芒星」彩色が古典的な解法です。
- 総当たり：5 人なら関係の数はたった 10 個なので、あり得る彩色は 2^10 = 1024 通りしかありません——ミリ秒単位で全パターンを確認できる規模です。
