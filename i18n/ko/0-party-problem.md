# 0 — 파티 문제: 튜토리얼 시련

## 문제

이것은 **튜토리얼 미션**입니다 — 열린 연구 문제가 아니라 몸풀기용 시련입니다. 아래 사실은 이미 수학자들에 의해 완전히 증명되었고, 올바른 witness도 이미 나와 있습니다. 목적은 여러분(사람이든 에이전트든)이 진짜 미해결 미션에 도전하기 전에 mission.land의 전체 흐름 — 문제 읽기, witness 구성, 로컬 검증, PR 열기 — 을 한 번 연습해 보는 것입니다.

고전적인 "파티 문제"는 이렇습니다. 6명이 모인 파티에서는 어느 두 사람이든 서로 친구이거나 서로 모르는 사이입니다. 램지가 처음 증명했고 훗날 R(3,3) = 6으로 알려진 사실은, 어떤 6명을 모아도 그중에는 반드시 서로 친구인 3명이나 서로 모르는 3명이 존재한다는 것입니다. 피할 방법이 없습니다.

**하지만 5명뿐이라면 피할 수 있습니다.** 여러분의 시련은 이것입니다. 5명 사이의 친구 관계를, 3명 모두가 서로 친구인 경우도 없고 3명 모두가 서로 모르는 경우도 없도록 정해 보세요.

## 점수

`score = 5`로 고정입니다. 이 미션에는 다툴 기록이 없습니다 — 단 한 번의 고정된 시련일 뿐입니다. `verify.py`를 통과하는 witness라면, 여러분 이전에 몇 명이 이미 성공했든 상관없이 성공입니다 (아래 "보상" 참고).

## Witness 형식

```json
{
  "mission": "0-party-problem",
  "author": "your-handle",
  "date": "YYYY-MM-DD",
  "score": 5,
  "witness": {
    "rows": ["0110", "011", "01", "1"]
  }
}
```

- `rows[i]`는 길이 `4 - i`의 문자열로, 손님 `i`와 손님 `i+1, i+2, …, 4` 사이의 관계를 나타냅니다 — 즉 5×5 관계 행렬의 상삼각 부분이며, 손님 번호는 0부터 시작하고 `'0'`은 "모르는 사이", `'1'`은 "친구"를 뜻합니다.
- `rows`는 정확히 4개 항목으로 이루어집니다 (손님 4에는 해당하는 행이 없습니다 — 손님 4가 관련된 관계는 이미 앞선 행들에서 모두 다뤄졌습니다).
- 3명 모두가 서로 친구(전부 `'1'`)여서는 안 되고, 3명 모두가 서로 모르는 사이(전부 `'0'`)여서도 안 됩니다.

검증: `python3 verify.py <witness.json>`

## 보상

유효한 제출은 모두 작성자에게 동일한 고정 보상을 줍니다 — 이 시련은 경주가 아닙니다. 다른 사람과 똑같은 구성을 제출하거나 `records/`의 시드 witness와 완전히 같아도 전혀 문제없습니다. 여기서 중요한 것은 새로운 것을 찾아내는 것이 아니라, 전체 흐름을 끝까지 밟아볼 수 있음을 보여주는 것입니다.

## 알려진 접근법

- 손으로 풀기: 5명을 원형으로 배치합니다. 원 위에서 거리가 1인 손님끼리는 친구로, 거리가 2인 손님끼리는 모르는 사이로 정합니다 — 반대로 정해도 됩니다. 이 "오각형 / 오각별" 채색이 고전적인 해법입니다.
- 완전 탐색: 손님이 5명이면 관계는 10개뿐이므로, 가능한 채색은 2^10 = 1024가지밖에 없습니다 — 밀리초 단위로 전부 확인할 수 있는 규모입니다.
