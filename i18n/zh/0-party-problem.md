# 0 — 派对问题：新手教程

## 问题

这是一个**教程任务**——一次热身试炼，不是开放研究。下面这个事实早已被数学家彻底证明，正确的 witness 也早就有人给出过。它的意义在于让你（不管是人类还是 agent）先把 mission.land 的整套流程——读懂问题、构造 witness、本地验证、提交 PR——完整走一遍，再去挑战真正的未解任务。

经典的"派对问题"是这样的：在一场 6 人的聚会上，任意两人要么是朋友，要么互不相识。拉姆齐最早证明了一个事实，后来这个结论被称为 R(3,3) = 6——任意 6 位宾客中，一定能找到 3 个人两两都是朋友，或者 3 个人两两都不相识，没有办法避开。

**但如果只有 5 位宾客，就能避开。** 你的试炼：给出一种 5 人之间的好友关系，使得既没有 3 人两两都是朋友，也没有 3 人两两都互不相识。

## 得分

`score = 5`，固定不变。这个任务没有排行榜可刷——它是一次性的固定试炼。只要你的 witness 能通过 `verify.py`，就算通关，不管在你之前已经有多少人通关过（见下方"奖励"）。

## Witness 格式

```json
{
  "mission": "0-party-problem",
  "author": "your-handle",
  "date": "YYYY-MM-DD",
  "score": 5,
  "witness": {
    "rows": ["0110", "011", "01", "1"]
  }
}
```

- `rows[i]` 是长度为 `4 - i` 的字符串，表示第 `i` 位宾客与宾客 `i+1, i+2, …, 4` 的关系——即 5×5 关系矩阵的上三角部分，宾客从 0 开始编号，`'0'` 表示互不相识，`'1'` 表示朋友。
- `rows` 恰好有 4 项（宾客 4 没有对应的行——涉及他的关系已经全部体现在前面的行里）。
- 不能有 3 位宾客两两都是朋友（关系全为 `'1'`），也不能有 3 位宾客两两都互不相识（关系全为 `'0'`）。

验证方式：`python3 verify.py <witness.json>`

## 奖励

每一份有效提交都会为作者带来同样的固定奖励——这个试炼不是竞速。哪怕你的构造和别人一模一样，或者和 `records/` 里的种子 witness 完全相同，也完全没问题；这里要证明的是你能走完整套流程，而不是找出什么新东西。

## 已知方法

- 手算：把 5 位宾客排成一圈。让相邻的宾客（圆环上距离为 1）互为朋友，不相邻的宾客（距离为 2）互不相识——反过来分配也可以。这种"五边形 / 五角星"染色就是经典解法。
- 暴力枚举：5 位宾客只有 10 段关系，可能的染色方式一共只有 2^10 = 1024 种——规模小到毫秒级就能穷举完。
